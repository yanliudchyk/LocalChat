## Введение
Данный документ содержит определения ключевых терминов, аббревиатур и технических понятий, используемых в техническом задании, архитектуре и исходном коде приложения **LocalChat**.

## 1. Роли и Сущности

| Термин | Определение |
|:---|:---|
| **Хост (Host / Server)** | Устройство, инициирующее создание чат-комнаты. Хост запускает `ServerSocket`, ожидает входящие подключения и выполняет роль маршрутизатора: принимает сообщения от одного участника и рассылает их всем остальным. |
| **Клиент (Client)** | Устройство, которое подключается к Хосту, используя известный IP-адрес и Порт. Клиент не имеет полномочий управлять сетью, его функции ограничены отправкой и приемом сообщений. |
| **Сессия (Session)** | Период активного взаимодействия между устройствами. Сессия начинается с успешного установления соединения (Handshake) и заканчивается разрывом связи (Disconnect) или закрытием приложения. |
| **Участник (Participant)** | Обобщенное название для любого пользователя чата, независимо от его технической роли (Хост или Клиент). |

## 2. Сетевые технологии

| Термин | Определение |
|:---|:---|
| **LAN (Local Area Network)** | Локальная вычислительная сеть. Окружение, в котором работает приложение: устройства подключены к одному сетевому оборудованию (роутеру) и видят друг друга напрямую, без выхода в интернет. |
| **IP-адрес** | Уникальный числовой идентификатор устройства в локальной сети (формат IPv4, например: `192.168.0.105`). Необходим Клиенту для обнаружения Хоста. |
| **Порт (Port)** | Числовой идентификатор (1024–65535), выделяемый операционной системой для конкретного процесса. Позволяет различать трафик LocalChat от трафика других приложений на устройстве. |
| **Socket (Сокет)** | Программный интерфейс («конечная точка»), обеспечивающий двунаправленный обмен данными. В коде соответствует классу `java.net.Socket`. |
| **ServerSocket** | Специальный тип сокета на стороне Хоста, который не используется для передачи данных, а слушает порт на предмет новых подключений. При подключении создает отдельный `Socket` для связи с конкретным клиентом. |
| **TCP (Transmission Control Protocol)** | Протокол передачи данных, обеспечивающий гарантированную доставку пакетов в правильном порядке. Используется в приложении для исключения потери частей сообщений. |
| **Broadcast (Бродкаст)** | Механизм рассылки, при котором сервер, получив сообщение от одного клиента, дублирует и отправляет его всем остальным активным подключениям (One-to-Many). |
